## Node.js 핵심 기능 데모 영역 분석

### 1. 상단 메뉴 탭의 의미
Node.js가 강력한 성능을 발휘하는 5가지 주요 활용 사례를 탭으로 구분해 두었습니다.

* **HTTP 서버 생성:** 가장 기본적인 웹 서버를 만드는 방법을 보여줍니다.
* **테스트 작성:** 내장된 테스트 러너를 사용하여 코드의 무결성을 검증하는 법을 보여줍니다.
* **파일을 읽고 해시합니다:** 파일 시스템(FS) 접근 및 암호화(Crypto) 기능을 통한 데이터 처리 방식을 보여줍니다.
* **스트림 파이프라인:** 대용량 데이터를 메모리 효율적으로 옮기는 Node.js의 핵심 기술인 **Stream(스트림)** 활용법을 보여줍니다.
* **스레드를 사용하여 작업하세요:** **Worker Threads(워커 스레드)**를 이용해 CPU 집약적인 작업을 병렬로 처리하는 법을 보여줍니다.

### 2. 현재 화면의 코드 분석 (HTTP 서버 생성)
이미지에 보이는 코드는 외부 프레임워크(Express 등) 없이 Node.js의 순수 내장 모듈만을 사용하여 웹 서버를 구축하는 코드입니다.

```javascript
// server.mjs
import { createServer } from 'node:http'; // (1)

const server = createServer((req, res) => { // (2)
  res.writeHead(200, { 'Content-Type': 'text/plain' }); // (3)
  res.end('Hello World!\n'); // (4)
});

// starts a simple http server locally on port 3000
server.listen(3000, '127.0.0.1', () => { // (5)
  console.log('Listening on 127.0.0.1:3000');
});
```

#### **[코드 상세 해설]**
* **(1) `node:http` 모듈 불러오기:** Node.js에 기본 내장된 HTTP 네트워크 프로토콜 기능을 가져옵니다. 최근 Node.js 표준에 따라 `node:` 접두사를 사용합니다.
* **(2) `createServer` 함수:** 클라이언트(웹 브라우저 등)로부터 요청이 들어왔을 때 어떻게 반응할지 정의하는 서버 객체를 생성합니다.
* **(3) `res.writeHead(200, ...)`:** 응답 상태 코드를 **200(성공/OK)**으로 설정하고, 전달할 데이터가 일반 텍스트(`text/plain`)임을 알리는 헤더를 작성합니다.
* **(4) `res.end()`:** 실제 데이터인 "Hello World!"를 클라이언트에 전송하고 응답을 마무리합니다.
* **(5) `server.listen(3000, ...)`:** 서버를 3000번 포트에서 대기 상태로 만듭니다. 이제 브라우저에서 `http://localhost:3000`으로 접속하면 결과를 볼 수 있습니다.

###  전문 용어 해설 (Terminology)

* **HTTP (HyperText Transfer Protocol, 하이퍼텍스트 전송 규약):** 웹 브라우저와 서버가 서로 데이터를 주고받기 위해 사용하는 통신 규약입니다.
* **MJS (Modular JavaScript, 모듈형 자바스크립트):** 자바스크립트의 최신 표준 모듈 방식(ES Modules)을 사용하는 파일 확장자입니다.
* **Port (포트):** 컴퓨터 내에서 실행되는 특정 프로세스(서버)에 접속하기 위한 논리적인 통로 번호입니다.
* **127.0.0.1 (Localhost, 로컬호스트):** 현재 사용 중인 컴퓨터 자신을 가리키는 IP 주소입니다.
* **Hash (해시):** 데이터를 고정된 길이의 고유한 문자열로 변환하는 기술로, 파일의 변조 여부를 확인할 때 주로 사용합니다.
* **Stream (스트림):** 데이터를 한 번에 모두 읽지 않고, 흐르는 물처럼 조금씩 나누어 처리하는 방식입니다. 대용량 파일 처리에 유리합니다.
